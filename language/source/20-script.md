# Project Neral · 自然演化文字规范 v2

## 核心立场

`Project Neral` 的文字不再按“概念图标集合”设计，而按“长期被真实书写共同体使用过的文字”设计。

这意味着它必须同时满足四件事：

1. **能区分**：高频字形在缩小、手写、低分辨率下仍能分辨。
2. **能快写**：日常书写应尽量控制在 `1–3` 个连续笔势内。
3. **能演化**：正式体、手写体、屏幕体之间允许出现系统性简化。
4. **能误写但仍可认**：轻微歪斜、连写、省笔后仍应保留识别核心。

## 字体类型学

这套文字属于“**语义前缀标记 + 音节链主写体**”。

- 一个完整实词通常由两个部分组成：
  - 前置的**语义标记**：提示语义域或词法角色。
  - 后续的**音节链**：以辅音骨架和元音附记记录读音。
- 语法词和极高频虚词允许省略语义标记，只保留音节链。
- 派生词允许更换语义标记，但音节链保持与转写系统一致。

## 书写层级

### 1. 正式体

- 用于碑铭、标题、站点主展示图。
- 笔画更完整，折点更清楚，保留字族结构。
- 强调字内重心和部件边界，便于首次学习。

### 2. 手写常用体

- 用于日常记录、草稿和快速书写。
- 允许角变圆、短横省略、相邻笔势连写。
- 高频字会优先向省力方向演化，而不是保留视觉装饰。

### 3. 屏幕体

- 用于网页与小尺寸显示。
- 以正式体为基准，但笔端略钝、内部留白更大。
- 小尺寸下优先保住轮廓，而不是追求细节。

## 构形原则

### 字形原子

基础字形围绕少量反复出现的“构形原子”展开：

- 主脊线：决定整体走势与书写方向。
- 开口角：负责区分字族与方向感。
- 挂钩：用于区分高频近形字。
- 短横：提供节奏和停顿点。
- 点记：只在需要时用于最小对立，不依赖大量点画。
- 杯形 / 环形：主要用于元音附记。

### 高频区分规则

- 高频辅音不能仅靠“有没有点”区分。
- 差异应优先来自：
  - 开口方向不同
  - 主脊长短不同
  - 挂钩位置不同
  - 横画所在高度不同
- 点记只作为最后一级区别特征。

### 语义标记规则

- 语义标记必须比音节链更稳定、更厚重。
- 它们像真实文字中的部首或分类前缀，应形成家族相似性。
- 高频语义标记控制在少量笔势内，避免每天都写“徽章”。

## 书写体验要求

- 单个辅音符平均 `2` 笔左右可完成。
- 元音附记尽量小，不抢主脊线轮廓。
- 词内阅读顺序清晰，从左到右推进，不制造过多回折。
- 相邻字之间留空，但字内部件紧凑，形成稳定词块。

## 自然演化痕迹

为了让这套文字像自然演化而来，`v2` 明确保留以下现象：

- 某些正式体折角在手写时圆化。
- 高频字的末端挂钩在快写时缩短或并入主脊。
- 元音附记在连续书写中允许轻微上移或下沉，但不改变类别。
- 同一家族字形共享共同骨架，只在一个核心位置发生差异。

## 混淆控制

当前最需要避免的混淆类型：

- `m / n`：不能只靠重复次数区分，还要在尾部形态上拉开。
- `t / d`、`s / z`：不能只靠单点区分，应同时改变停笔位置。
- `l / r`、`y / w`：需要在末端回转或分叉结构上明显区分。
- 语义标记与普通音符不能共享主轮廓。

## 网页交付规则

- 页面继续同时展示：原生字形、标准转写、IPA、中文释义。
- SVG 资产以正式体为主，同时在字形元数据中保留手写体提示。
- 字形资源加载失败时，回退到“简写记号 + 标准转写”，而不是只显示抽象图标。

## 当前阶段边界

- `v2` 的目标是先让基础字集看起来像“真的能写”，不是立刻完成整套字体工程。
- 在正式进入 `WOFF2` 之前，必须先对基础字集完成一轮：
  - 高频近形检查
  - 小尺寸检查
  - 手写省笔检查
  - 连续词块检查
