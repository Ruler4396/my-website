# Project Neral · 设计简报

`Project Neral` 是这门语言的内部代号，不是最终语言名，也不是最终文字名。

## 目标

- 创造一门完全原创、可持续扩展、可用于日常表达与短文写作的新语言。
- 优先保证“可学、可写、可检索、可在网页上稳定展示”，而不是只做视觉设定。
- 保留自然演化感：允许少量不对称和语义漂移，但整体上仍然规整、适合教学。

## 用户与使用场景

- 第一使用者是站点作者本人，需要能持续扩词、写短文、记日志、做页面展示。
- 第二使用者是访客，需要在不安装字体、不下载输入法的前提下理解其读音和基础结构。
- 第一版网站只承担展示、检索、阅读和审阅，不承担正式学习闭环。

## 审美边界

- 声音：清晰、克制、偏明亮，不依赖极端发音制造陌生感。
- 文字：采用“语义标记 + 音符链”的表意混合文字，字面上就能提示词类和语义域。
- 语法：以分析语为主，辅以少量派生黏着，避免大规模屈折。

## 设计原则

1. 任何原生词必须有唯一标准转写。
2. 词库增长必须服从构词系统，不靠随机起词。
3. 高用法优先于高设定：先覆盖自我介绍、时间地点、需求表达、说明性文本。
4. 网页展示优先使用稳定 SVG 字形，等字符集收敛后再做字体化。

## 明确不做

- 当前阶段不做音频与 TTS。
- 当前阶段不做正式输入法与 Unicode 编码方案。
- 当前阶段不把语言内容塞进 `content/` 文章索引，也不修改主站首页信息架构。

## V1 完成标准

- 一套冻结的音系、音节结构、重音规则和标准转写。
- 一套可解释的文字书写逻辑和基础字形资产。
- 一套能表达日常和说明场景的核心语法。
- `500–800` 条词典词条。
- `50–100` 条例句。
- `3` 篇短文本样本。
- 一个可在 `MyWeb` 里直接访问的临时评审页。
